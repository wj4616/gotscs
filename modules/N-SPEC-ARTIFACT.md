## Review-gate hop note

## Protocol

0. **Read briefing-core.md (and the appendices declared in the per-node read-map below).**
   <!-- DD-03 read-map for N-SPEC-ARTIFACT: briefing-core only. No appendices required. -->

1. **Read inputs.**
   - Read `stages/N-SYNTH-GRAPH.md` (graph_spec — primary).
   - Read `stages/N-AGG-DESIGN.md` (design_blueprint — for skill_name, input/output shapes, success criteria).
   - Read `stages/N-NORMALIZE.md` (domain, input_shape, output_shape, success_criteria, latency_tolerance).
   - Read `stages/N-CONSTRAINTS.md` (inventory_items, anti_patterns_guarded — for Hard Constraints section).
   - If `stages/N-CONTEXT-ANALYZE.md` exists: read it for the Design Decisions section.

2. **Determine output paths.**
   - Resolve skill_name slug from `domain` field of normalize_digest. **slugify rules** (canonical, identical to N-EMIT step 3):
     1. Lowercase the input string.
     2. Replace any character outside `[a-z0-9-]` with `-`.
     3. Collapse runs of consecutive `-` into a single `-`.
     4. Strip leading and trailing `-`.
     Example: `"Code-Review Summarization!"` → `code-review-summarization`.
   - Final user-facing path: `<session_output_base>/<skill_name>-spec.md`.
   - Stage file path: `<session_dir>/stages/N-SPEC-ARTIFACT.md` (HC-21 edge-coverage compliance).

3. **Render the spec document** with all required top-level sections in order:

   ```markdown
   ---
   skill_name: <sanitized domain slug>

   topology: <topology_class from N-TOPOLOGY>

   wave_count: <total_waves from graph_spec metadata>
   input_type: <input_shape>
   output_type: <output_shape>

   > Generated by GOTSCS v4. Feed to `/writing-plans` for a task-by-task implementation plan, or use the Standalone Prompt section to build the skill directly in a new conversation.

   <2-4 sentences: what this skill does, who uses it, what problem it solves. Derived from domain + success_criteria. Written for a human reader.>

   ## Input / Output
   **Accepts:** <input_shape — verbatim from normalize_digest>
   **Produces:** <output_shape — verbatim from normalize_digest>

   ## Success Criteria
   <Bulleted list of success_criteria verbatim from normalize_digest.>

   ## Hard Constraints
   <Bulleted list of inventory_items verbatim from constraints_digest. These MUST appear verbatim in any implementation's Hard Gates.>

   ## Pipeline Architecture
   **Topology:** <topology_class> | **Waves:** <N> | **Nodes:** <N>

   ### Topology Diagram

   ## Aggregation Architecture

   ## Wave Plan

   ```

   When invoked with `--context`, additionally render any Design Decisions context-classification rows (preservation-candidate / upgrade-target / replace-target) with one-sentence rationale; replace-target rationales must cite the specific structural defect. Merge into the required Design Decisions section above.

3.5. **Post-render completeness check (spec-mode quality gate).**
   <!-- GAP-003 fix: N-VERIFY is mode-gated to skill/both; this step closes the spec-mode verification gap. -->
   After rendering and before any disk write, perform three inline checks:

   **(a) Section completeness.** The 13 required top-level headings are:
   `Purpose`, `Input / Output`, `Success Criteria`, `Hard Constraints`, `Pipeline Architecture`,
   `Node Specifications`, `Aggregation Architecture`, `Wave Plan`, `Per-Wave Token Cost`,
   `Design Decisions`, `Failure Modes`, `Implementation Notes`, `Standalone Prompt`.

   For each: scan the rendered text for `## <section>`. Accept both `"Input / Output"` and
   `"Input/Output"` as equivalent matches for that entry.

   - **0 sections missing:** continue to step 4.
   - **1–3 sections missing:** insert each missing section with a one-sentence placeholder:
     `> [auto-inserted during completeness check — populate before use]`
     Prepend `standalone_prompt_repaired: true` to the rendered frontmatter as a flag.
     Continue to step 4.
   - **>3 sections missing:** do NOT write to disk. HALT with advisory:
     `"N-SPEC-ARTIFACT: {count} required sections absent after render — retry step 3."`

   **(b) Standalone Prompt word count.** Count words in the block between `<role>` and `</constraints>`
   (inclusive of tag text).
   - **≥100 words:** continue.
   - **<100 words:** expand the `<context>` sub-block with additional pipeline architecture context
     until ≥100 words. Re-count. Set `standalone_prompt_expanded: true` in rendered frontmatter.

   **(c) Standalone Prompt sub-block presence.** Verify all four tags exist:
   `<role>`, `<context>`, `<task>`, `<constraints>`.
   - If any missing: insert a minimal well-formed placeholder for the absent tag.
     Set `standalone_prompt_repaired: true` in rendered frontmatter.

4. **Dual-write contract (per F011 fix).**
   - Write the rendered document to `stages/N-SPEC-ARTIFACT.md` (intermediate stage file for HC-21 edge-coverage compliance).
   - Write the same content to `<session_output_base>/<skill_name>-spec.md` (final user-facing artifact).
   - Both files MUST contain identical content.

5. **Sync spec_path to SIGNAL_STATE.json.**
   <!-- GAP-002 fix: was bare jq > /tmp/_ss.json (race-prone hardcoded path, diverged from P-005). -->
   <!-- Now uses sync-signal.sh for atomic write + pipefail semantics, consistent with all other nodes. -->
   ```bash
   ~/.claude/skills/gotscs/scripts/sync-signal.sh "$SESSION_DIR" spec_path="<session_output_base>/<skill_name>-spec.md"
   ```

6. **Emit signals.**
   - `spec_document`: the rendered markdown content
   - `spec_complete`: true (after both writes succeed)

7. **Mode-conditional exit (E21 terminal).** When `MODE=spec`: orchestrator fires terminal edge E21 → SPEC_OUTPUT after this node returns. Pipeline exits cleanly without entering Waves 9-10. Print to user:
   ```
   ## GOTSCS Spec Complete

   Spec written to: <session_output_base>/<skill_name>-spec.md

   To generate an implementation plan:
     /writing-plans <session_output_base>/<skill_name>-spec.md

   To build the skill directly: paste the Standalone Prompt section into a new conversation.

   To build the full skill package instead:
     gotscs --skill "<your brief>"
   ```

   When `MODE=both`: do not exit; orchestrator fires E22/E23 chaining `spec_complete=true` into Wave 9.

8. **Review-gate pause hook (when `--review-gates` set).** After step 4 dual-write completes: the orchestrator suspends pipeline execution, surfaces the rendered spec to the user, and awaits explicit approval before continuing. On user-rejection: orchestrator routes back to N-AGG-DESIGN with the user's rejection note as remediation_payload (subject to retry_count cap).

## Scale gates
- tokens: 6000
- time: 480s
- spawns: 0 (inline)
- retries: 1 (single retry on render failure; on second failure HALT with `halt-on-spec-render-failure` advisory)

## Failure modes
- timeout: retry once; on second timeout HALT with `halt-on-spec-render-failure`
- malformed output (any of the required sections missing): retry once; if still missing emit advisory listing the missing sections
- missing input: HALT "N-SPEC-ARTIFACT: graph_spec or design_blueprint missing"
- format-mismatch on Edge: re-read stages/N-SYNTH-GRAPH.md and stages/N-AGG-DESIGN.md directly

## Graceful Degradation
After retry exhaustion, emit a degraded output with the following frontmatter prefix:
```yaml
degradation_notice:
  node_id: N-SPEC-ARTIFACT
  retry_count: <n>
  last_error_class: timeout | malformed | missing_input | subagent_crash
  degraded_fields: [<field1>, <field2>]
  confidence: low | medium
```
Pipeline does NOT halt. Orchestrator propagates degradation_notice to N-VERIFY (informational).
